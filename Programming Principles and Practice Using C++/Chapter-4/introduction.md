# 💣 3. Dangerous Errors (Book Highlight 🔥)

👉 Two worst types:

---

## ❌ 1. Undetected logic error

```cpp
int a = 5;
int b = 2;
std::cout << a / b;  // 2 (wrong expectation)
```

👉 program works ❗
👉 result wrong ❗

---

## ❌ 2. Requirement mismatch

👉 User wants:

```text
calculate age correctly
```

👉 Program does:

```text
wrong calculation
```

👉 not technical error — but **real-world failure**

---

# 🧠 4. Important Question

👉 Book கேட்கும்:

> “Is my program supposed to detect that error?”

---

## Example:

👉 power cut ⚡

* simple program → ignore ✔️
* medical system → must handle ❗

---

# 🎯 5. Assumptions (VERY IMPORTANT)

👉 Book rules:

---

## ✔️ Your program should:

1. **Correct output for valid input**
2. **Error message for invalid input**

---

## ❌ Your program need NOT:

3. handle hardware failure
4. handle OS failure
5. continue after fatal error

---

👉 This = **basic professionalism** 💯
