# 🧠 Core Idea

👉 Grammar rules இருந்தது:

```text id="5h7mco"
Expression → + -
Term       → * / %
Primary    → number, ()
```

👉 Now:

```text id="v7rr3h"
Each rule = one function ✔️
```

---


# 🟢 2. Function Responsibilities


## 🔹 expression()

```text id="6j4l6d"
+ -
```

👉 calls `term()`

# 🟢 4. BIG BREAKTHROUGH 🔥

👉 Book சொல்றது:

# 🟢 5. Why get_token() different?

👉 expression(), term(), primary():

✔ return result (double)

---

👉 get_token():

```text id="ng0xqz"
returns Token (not value)
```

---

## 💡 Tamil

👉 because:

```text id="0bbp36"
+ , ( , * → value இல்லை
```

---

# 🟢 6. Final Function Design

```cpp id="k6ex5y"
Token get_token();
double expression();
double term();
double primary();
```

---

# 🧠 Flow

```text id="9x0u3d"
expression()
  → term()
     → primary()
```

---

# 🟢 7. Execution Example

Input:

```text id="fgsq41"
1 + 2 * 3
```

---

👉 Flow:

```text id="7c1x4y"
expression()
  → term()
     → primary() → 1
  → + term()
        → primary() → 2
        → * primary() → 3
```

---

👉 Result:

```text id="pg02b3"
1 + (2*3) = 7 ✔️
```

---

# 💥 8. Why this design works?

👉 Because:

```text id="46cwqs"
Hierarchy = precedence
```

---

# 🧠 Key Insight

```text id="qz1n7x"
Primary < Term < Expression
```

👉 lower → higher
