ஆம் 👍 — இரண்டு இடங்களிலும் **object உருவாகும்**. ஆனா என்ன exactly நடக்குது என்பதை தெளிவாகப் பார்ப்போம்.

# 🔥 4. Important: Copy / Move / Optimization

👉 3 possibilities:

---

## 🟡 (Old way)

```text
temporary → copy → t
```

👉 extra copy ❌

---

## 🟢 (Move semantics)

```text
temporary → move → t
```

👉 faster ✔️

---

## 🚀 (Modern C++ - RVO / NRVO)

👉 compiler optimize பண்ணும்:

```text
temporary create கூட பண்ணாது ❗
directா t-ல construct பண்ணும் ✔️
```

👉 இதை சொல்வது:

```text
Return Value Optimization (RVO)
```

# 🧠 Real behavior (modern compilers)

```text
Token t directly உருவாகும்
```

👉 intermediate object இல்லை
