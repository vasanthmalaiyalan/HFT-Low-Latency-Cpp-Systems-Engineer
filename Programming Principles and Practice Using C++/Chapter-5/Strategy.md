# 🧠 Core Idea

👉 Programming = strategy + thinking

👉 Book gives **4-step strategy**

```text
1. Understand problem
2. Break into parts
3. Build small version
4. Build full solution
```

---

# 🟢 [1] Understand the Problem (VERY IMPORTANT)

👉 Book says:

> Be specific about what you are trying to do 

---

## 💡 Tamil Explanation

👉 First கேள்வி:

* “இந்த program என்ன செய்யணும்?”
* “User எப்படி use பண்ணுவான்?”

👉 IMPORTANT:

* “HOW” பற்றி யோசிக்காதே ❌
* “WHAT” பற்றி மட்டும் யோசி ✔️

---

## 🧪 Example (Calculator)

👉 Wrong thinking ❌:

```text
loop use பண்ணலாமா? function எழுதலாமா?
```

👉 Correct thinking ✔️:

```text
User input என்ன?
Output எப்படி இருக்கணும்?
```

---

## ⚠️ Problem clarity

👉 Book சொல்றது:

> Problem statement clearஆ இருக்காது

👉 So:

* clarify பண்ணு
* assumptions check பண்ணு

---

## ⚠️ Don’t be greedy 😅

👉 பெரிய mistake:

👉 “super advanced calculator build பண்ணணும்”

👉 Book advice:
✔️ small start பண்ணு

👉 later:
✔️ version 2.0 build பண்ணலாம்

---

## ⚠️ Feasibility

👉 உன்னால முடியுமா?

* time இருக்கா?
* skill இருக்கா?

👉 இல்லனா:
✔️ problem simplify பண்ணு

---

# 🟢 [2] Break into Parts

👉 Book says:

> Divide program into manageable parts 

---

## 💡 Tamil Explanation

👉 பெரிய problem → small pieces

---

## 🧪 Calculator Example

```text
1. Input read
2. Token split
3. Expression parse
4. Calculate
5. Output
```

---

## 🔧 Libraries use பண்ணு

👉 Book சொல்றது:

* reinvent the wheel வேண்டாம் ❌

👉 Use:

* standard library (`iostream`, `vector`, etc.)

---

## 🧠 Car Example (Book)

👉 Car build பண்ணும்போது:

* wheel
* engine
* seat

👉 separately build பண்ணுவோம்

👉 same → program

---

## 💡 Reusable Parts

👉 Code parts reusable ஆக இருக்கணும்

Example:

* `Token`
* `Token_stream`

---

# 🟢 [3] Build Small Version (Prototype)

👉 இது மிக முக்கியம் 🔥

👉 Book சொல்றது:

> First version = small + limited 

---

## 💡 Tamil Explanation

👉 First attempt:

* full program வேண்டாம் ❌
* small version மட்டும் ✔️

---

## 🧪 Example

👉 First version:

```text
Only + operator handle பண்ணு
```

👉 later:

```text
*, / add பண்ணு
```

---

## 🔁 Why?

👉 To:

* find mistakes
* understand problem better
* test ideas

---

## 💥 Important Truth

👉 First version:

❌ ugly
❌ wrong

👉 That’s NORMAL

---

## 🔄 Prototype concept

👉 First version = prototype

👉 If bad:
✔️ throw away
✔️ rewrite

👉 Book சொல்றது:

> Don’t continue with messy code ❌ 

---

# 🟢 [4] Build Full Solution

👉 Now final program build பண்ணு

---

## 💡 Tamil Explanation

👉 Use:

* working parts
* tested logic

---

## ⚠️ Don’t do this ❌

👉 One shot full program எழுதாதே

👉 Why?

* errors அதிகம்
* debugging nightmare

---

## ✔️ Correct approach

```text
small working parts → combine → full program
```
